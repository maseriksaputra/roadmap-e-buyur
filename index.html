🎯Definition of Done (Target Presentasi)

Sasaran Akhir – “Siap Demo Awal Oktober”
  • Backend Laravel di Hypercloud (domain + HTTPS Let’s Encrypt), APP_URL benar.
  • DB migrasi berjalan; backup harian otomatis + verifikasi restore.
  • Aplikasi Flutter buyer & seller terhubung (base URL via --dart-define).
  • Auth & role buyer/seller beres; routing dashboard sesuai role.
  • ML TFLite on-device: klasifikasi jenis + % kelayakan; threshold default 70% (kalibrasi 65–80%).
  • Batch Scan: scan beruntun → draf lokal → commit massal.
  • Gambar produk (Storage::url + symlink), cart & fav per-user (limit 100), search buyer global & seller scoped, “Kelola Produk/Toko” ter-scope seller.
  • Stabil: session DB + Sanctum + CORS.
  • QA basic alur utama + materi presentasi & runbook deploy/recovery.
Hardening: APP_KEY, rate limit login, exception logging aktif.

🧩Perbaikan Non Teknis

Buyer UI/UX & Flow

Beranda

  • Perbaiki overflow pada product card (bottom overflowed by 3.9px).
  • Perbaiki bug pencarian.
  • Detail produk: tambah tampilan ulasan (posisi di atas rekomendasi produk lain).

Pencarian

  • UI hasil pencarian dibedakan dari beranda.
  • Filtering lebih maksimal: jenis, harga, rentang % kelayakan, jarak.

Keranjang

  • Navigasi ke checkout: validasi pembelian, alamat, pembayaran, kurir.
  • Poles tampilan keranjang (hierarki & CTA).

Riwayat Pembelian

  • Integrasi status: diproses → dikirim → selesai (by id).
  • Setelah selesai: tombol beri ulasan & beli lagi.
  • Detail riwayat pembelian dengan roadmap status.

Profil Buyer

  • Tampilkan identitas dari DB by user id (nama, email, no HP, dll.).
  • Tampilkan kuantitas keranjang & favorit (by id) di profil.
  • Hilangkan rating di profil.
  • Perbaiki navigasi edit profil.
Seller Operasional Toko

Beranda

  • Ringkas pendapatan 1 bulan terakhir.
  • Kuantitas pesanan realtime ke DB (stok & produk by seller id).
  • Kesegaran: rata-rata kelayakan produk.
  • Produk terbaru (by seller id).

Kelola Produk

  • Total inventori (∑ harga × stok).
  • Perbaiki filtering.
  • Integrasi kelola produk by id.
  • Detail & edit: stok, nama, nutrisi, deskripsi.
  • Hapus produk.

Scan Produk

  • Perbaiki animasi analyzing.
  • Tambah rincian & keterangan analisis.
  • Integrasikan prediksi objek & kualitas (persentase + label kelayakan).
  • Create Product: atribut “Kandungan Nutrisi” dengan enumerasi nutrisi umum (opsi custom).

Riwayat Penjualan

  • Status: orderan, diproses, selesai, dibatalkan.
  • Detail pesanan: konfirmasi, terima, teruskan pesanan.

Profil Seller

  • Identitas by seller id (identitas pribadi & toko).
  • Tampilkan identitas di profil seller.
